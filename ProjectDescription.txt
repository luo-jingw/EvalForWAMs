LingBot-VA × ViDiT-Q 量化项目 — 高层交接说明

  项目本质

  对 LingBot-VA（5.09B params bf16 WAN transformer）做 ViDiT-Q 量化（W8A8 + W4A8），在 RoboTwin 15 任务上评估 success rate / GPU 显存峰值 / 推理时延，做 bf16 baseline
  三方对比。评估性研究项目，要求忠实复刻 ViDiT-Q 方法，不私自简化。

  - Conda env RoboTwin-jw（client + sapien 仿真）

  整体架构（M / Q 模块编号在 plan.txt 中）

  评估管道（method-agnostic, Part I）：
    M1 perf_probe     - PerfProbe 类，CUDA Event 时延 + peak memory，写 JSONL
    M2 server         - wan_va_server.py，5 阶段插桩 (init/text_encoder/vae_encode/transformer/action_head)
    M5 aggregator     - calc_perf_stat.py，JSONL + RoboTwin SR → summary.csv/json
    M6 orchestrator   - run_eval.sh，3 模式 (smoke/single/pool)，pool 是 worker-queue + flock + VRAM 等待
    M7 monitor        - monitor.sh 实时 dashboard

  量化变体（plug-in, Part II/III, 通过 importlib 加载）：
    Q1-Q11  - configs / ptq / kernel / linear modules / block / loader / cross_ckpt 等
    Server  - 用 --variant viditq --variant_args <yaml> 切换

  关键约定

  1. 量化方法：per-channel asymmetric weight（int8 + int16 zero-point）+ per-token symmetric activation（与 ViDiT-Q W8A8/W4A8 config 一致）。
  2. CUDA kernel 来源：完整拷贝 ViDiT-Q 的 w8a8_gemm_cuda.cu 和 w4a8_per_channel_gemm_cuda_qserve.cu，仅做一处扩展：加 typename OutT 模板参数 + DtypeTraits<__nv_bfloat16>
  特化，让 ViDiT-Q 原本只支持 fp16 输出变成 fp16 + bf16 双支持。其他模板分支（weight_asym, has_bias, ScaleMulMode, OutputDtype, K_STAGE, CTA tile）一律保留。
  3. fp16 launcher 保留：作为对照实验，bench 时与 bf16 数值对比。
  4. 修改 upstream 必备份：所有改过的 upstream 文件留 .bak。
  5. 任务集：15 个 RoboTwin 任务（1 long + 4 medium-long + 10 short），每任务 25 episode。
