LingBot-VA × ViDiT-Q 量化项目 — 高层交接说明

  项目本质

  对 LingBot-VA（5.09B params bf16 WAN transformer）做 ViDiT-Q 量化（W8A8 + W4A8），
  在 RoboTwin 15 任务上评估 success rate / GPU 显存峰值 / 推理时延，做 bf16 baseline
  三方（最终 4-way 含 static act）对比。评估性研究项目，要求忠实复刻 ViDiT-Q 方法，
  不私自简化。

  Server 通过 --variant viditq --variant_args <yaml> 切量化路径。Loader 解析路径模板：
    importlib.import_module(f"ptqeval.wam.{wam_name}.method.{method_name}.loader")
  加新方法只需在 ptqeval/wam/<wam>/method/<new>/ 下加 loader.py 等，server 无需修改。

  关键约定

  1. 量化方法：per-channel asymmetric weight（int8 + int16 zero-point）+
     per-token symmetric activation（与 ViDiT-Q W8A8/W4A8 config 一致）。
     注意：activation 当前是 **dynamic per-token**（每次 forward 重算 scale），
     ViDiT-Q 默认 image-DiT 配方是 **static per-channel from calibration**。
     Part V (Phase 31-36) 补 static 路径形成 3-way 对比。
  2. CUDA kernel 来源：整文件 verbatim 拷贝 ViDiT-Q 的 w8a8_gemm_cuda.cu，仅做一处扩展：
     加 typename OutT 模板参数 + DtypeTraits<__nv_bfloat16> 特化，让 ViDiT-Q 原本只支持
     fp16 输出变成 fp16 + bf16 双支持。其他模板分支（weight_asym, has_bias, ScaleMulMode,
     OutputDtype, K_STAGE, CTA tile）一律保留。
  3. fp16 launcher 保留：作为对照实验，bench 时与 bf16 数值对比。
  4. 第三方仓库工作树 == upstream（lingbot-va/.git_orig HEAD）。所有研究代码
     在 PTQEval/。修改 upstream 是反模式；新功能 fork 进 PTQEval。
  5. 任务集：15 个 RoboTwin 任务（1 long + 4 medium-long + 10 short），每任务 25 episode。
  6. P5 commit 风格：每 milestone 单句 commit message（参考 `git log --oneline`）。
